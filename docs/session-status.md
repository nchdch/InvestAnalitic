## 2. Что сделано в этой сессии: правки заказчика (3 пункта) + UI организаций

Заказчик попросил три правки в карточках портфеля:

1. **Количество бумаг** — во всех таблицах портфеля (сводный отчёт «Портфель», «Все
   активы», вкладки «Акции»/«Облигации») теперь показывается `Кол-во` по каждой позиции
   (`position.quantity`).
2. **Доля от категории актива и от всего портфеля** — добавлено поле
   `assetTypeWeight` в `EquityRow`/`BondRow` (`src/types/portfolio.ts`), считается на
   сервере в `getPortfolioSummary()` (`server/src/services/portfolioService.ts`) как
   `currentValue / totalEquityValue` (или `totalBondValue`) × 100. Во всех таблицах
   позиция показывает и `% от категории` (assetTypeWeight), и `% от портфеля`
   (portfolioWeight). На уровне групп «Акции»/«Облигации» — суммарная доля категории
   (сумма assetTypeWeight по строкам).
3. **FIFO/LIFO для налогового учёта** — `useRealizedPnl.ts`: функция `computeFifo`
   обобщена до `computeLots(trades, year, method)`, для LIFO лоты списываются с конца
   очереди (`queue.splice(queue.length-1, 1)`). Метод хранится в `useSettingsStore`
   (`taxLotMethod: 'FIFO' | 'LIFO'`, по умолчанию FIFO, персистится в localStorage),
   переключатель — в Профиле, секция «Налоговый учёт». `AnalyticsPage` показывает
   активный метод в подзаголовке карточки «Реализованный P&L».

Дополнительно (было не закоммичено с предыдущей сессии, задеплоено сейчас вместе):

4. **Карточка «Организации» в профиле** — список участников орг., роли, заявки на
   вступление, приглашение по email (`OrgMembersModal.tsx`, новый файл).
5. **`usePortfolio`** — фоновое обновление котировок (`bump()`) больше не дёргает
   спиннер и не сбрасывает состояние при повторных загрузках одной организации
   (`loadedOrgRef`).

   сих пор не задан (см. предыдущие статусы). `/api/assistant/chat` отвечает SSE
   `event: error`. Нужно выбрать провайдера (с учётом гео-блокировок OpenAI из РФ —
   `deepseek`/`openrouter`/прокси), прописать `LLM_PROVIDER`/`LLM_API_KEY`/`LLM_MODEL`
