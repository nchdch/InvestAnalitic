## 3. Известные проблемы / TODO для следующей сессии

1. **AI-ассистент задеплоен, но без рабочего провайдера** — `LLM_API_KEY` на проде до
   сих пор не задан. `/api/assistant/chat` отвечает SSE `event: error`. Нужно выбрать
   провайдера (с учётом гео-блокировок OpenAI из РФ — `deepseek`/`openrouter`/прокси),
   прописать `LLM_PROVIDER`/`LLM_API_KEY`/`LLM_MODEL` в корневой `.env` на проде,
   `docker compose -f docker-compose.prod.yml up -d` (без `--build`).

   **Локальная LLM в Docker на проде — не вариант на текущей VM**: проверено
   2026-06-15 — `10.3.0.47` имеет всего 2 vCPU, ~2,9 ГБ свободной RAM и ~5,9 ГБ
   свободного диска (без GPU). Даже квантованная 7-8B модель (Hermes/Qwen и т.п.)
   не поместится рядом с Postgres/backend/frontend/nginx и будет работать очень
   медленно. Пользователь попросил отдельно обсудить **увеличение ресурсов прод-VM**
   перед тем, как пробовать локальную LLM — этот разговор предстоит в следующей сессии,
   пока ничего не менялось.

2. **`accounts.user_id IS NULL`** — счета без `user_id` доступны всем авторизованным
   пользователям (fallback в `getAccessibleAccountIds`). Нужен одноразовый backfill
   после подтверждения владельцев.

3. **node/npm/npx/docker/psql недоступны локально** — typecheck/build/test только через
   прод-сборку по SSH.

4. В корне репозитория untracked PNG-скриншоты (`Скриншот*.png`, `Снимок*.png`) —
   не закоммичены, не в `.gitignore`.

5. `.claude/scheduled_tasks.lock` помечается изменённым почти каждую сессию —
   автогенерируемый файл, коммитить не нужно.

6. Не реализовано (карточка «Что пока не учитывается» на странице Аналитика): P&L за
   период (день/неделя/месяц/квартал/год) — нет исторических снимков портфеля; карта
   по секторам; топ движений рынка вне портфеля; форвардная дивидендная доходность.
